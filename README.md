# tgif.sh

Landing page & Config Pilot for [Friday](https://github.com/tgifai/friday) — a Go-based multi-agent AI assistant framework.

Built with React 19, Chakra UI v3, TypeScript, and Vite.

## Features

- **Landing Page** — Hero, feature showcase, quick-start guide, and integration logos
- **Config Pilot** (`/pilot`) — Interactive 5-step wizard that generates a valid `config.yaml` for Friday
  - General settings (gateway & logging)
  - LLM provider setup (OpenAI, Anthropic, Gemini, Ollama, Qwen)
  - Channel configuration (Telegram, Lark, HTTP)
  - Agent wiring (model selection, channel binding, runtime params)
  - YAML preview with copy & download
- **i18n** — English / Chinese, auto-detected from browser
- **Dark Mode** — Powered by `next-themes` via Chakra UI

## Tech Stack

| Layer | Tech |
|-------|------|
| Framework | React 19 + TypeScript 5.9 |
| Build | Vite 7 |
| UI | Chakra UI v3 + Emotion |
| State | MobX (Config Pilot) |
| Routing | React Router v7 |
| i18n | i18next + react-i18next |
| YAML | js-yaml |

## Getting Started

```bash
# Install dependencies
npm install

# Start dev server
npm run dev

# Production build
npm run build

# Preview production build
npm run preview

# Lint
npm run lint
```

## Project Structure

```
src/
├── main.tsx                    # Entry point (providers, router)
├── App.tsx                     # Route definitions
├── components/
│   ├── Navbar.tsx              # Navigation bar
│   ├── Footer.tsx              # Site footer
│   ├── LanguageSwitcher.tsx    # EN / ZH toggle
│   └── ui/                    # Chakra UI snippets (auto-generated)
├── i18n/
│   ├── index.ts               # i18next config
│   └── locales/
│       ├── en.json
│       └── zh.json
└── pages/
    ├── Landing/               # Home page
    │   └── components/        # Hero, Features, QuickStart, Integrations
    └── Pilot/                 # Config wizard
        ├── index.tsx          # Page shell + stepper
        ├── yaml.ts            # State → YAML conversion
        ├── validation.ts      # Per-step validation
        ├── store/
        │   ├── PilotStore.ts  # MobX store
        │   ├── context.tsx    # React context provider
        │   ├── types.ts       # TypeScript interfaces
        │   └── defaults.ts    # Default values & field definitions
        └── steps/
            ├── GeneralStep.tsx
            ├── ProvidersStep.tsx
            ├── ChannelsStep.tsx
            ├── AgentsStep.tsx
            └── ReviewStep.tsx
```

> **Note:** Files in `src/components/ui/` are auto-generated by `@chakra-ui/cli`. Do not edit them manually. Regenerate with:
> ```bash
> npx @chakra-ui/cli snippet add --all --force
> ```
